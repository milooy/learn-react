## 컴포넌트를 얼마나 쪼개야 할까?

- 하나의 컴포넌트에 모든 기능 넣어도 문제 없이 돌아간다.
- 하지만 여기저기 가져다 쓰기 위해 컴포넌트를 쪼갠다.
- 기타 장점
  - 가독성(관심사별로 코드가 나뉘어있으니)
  - 잠재적 버그 줄이기(컴포넌트를 추가하거나 삭제해도 다른 UI에 지장이 없음)

## 컴포넌트 데이터의 2가지 타입: props / state

컴포넌트를 개발할 때는 들어가는 데이터를 2가지 타입으로 나누기

#### 컴포넌트를 쓸 때 주입하는 데이터: `props`

- 컴포넌트를 사용하는 상황에 따라 넘겨줄 적절한 데이터
- e.g. 첫번째 컴포넌트엔 고양이A사진 넘기기, 두 번째 컴포넌트엔 고양이B사진 넘기기

#### 컴포넌트 내부에서 관리하는 데이터: `state`

- 사용자 인터렉션에 따라 변하는 데이터. 외부에는 노출X
- e.g. 하트를 누르면 빨간색으로 보여주는 기능을 컴포넌트 내부에서 관리
